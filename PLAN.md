# Plan: Spin Up Agent Session for Safety-Net Commits

## Problem

When implementation succeeds but the agent leaves uncommitted changes in the worktree, `commitAndPushRepos` in `internal/orchestrator/implementation.go` (line 1287) does a hardcoded safety-net commit:

```go
gitStageAndCommit(ctx, sess.WorktreePath, "chore: commit residual changes before push")
```

This ignores the user's configured `commit.message_format` setting (`ai-generated`, `conventional`, `custom`) and `commit.message_template`. The result is a generic commit message that provides no signal about what actually changed.

## Approach

Instead of committing with a hardcoded message, spin up a brief agent session whose sole job is to produce and apply a commit. The agent receives:

1. The diff of the uncommitted changes (`git diff --stat` + `git diff`)
2. The configured commit message format settings as a prompt section

This reuses the existing `AgentHarness.StartSession` path, so the commit message is generated by the same agent that did the implementation work, respecting whatever format conventions the user configured.

## Changes

### 1. Add `buildCommitMessagePrompt` to `internal/orchestrator/implementation.go`

A new function that builds the system prompt for the commit-only session. It includes:

- A task description: "You are committing residual changes left in the worktree after an implementation session."
- The output of `git diff --stat` and `git diff` for the uncommitted changes
- A "Commit Message Format" section derived from the user's config:

  - **`ai-generated`**: "Write a concise, descriptive commit message summarizing what changed and why. Use imperative mood."
  - **`conventional`**: "Write a commit message using Conventional Commits format (type(scope): description). Choose the appropriate type (feat, fix, refactor, docs, test, chore, etc.)."
  - **`custom`**: Include the user's `message_template` value verbatim as the template to follow.

- An instruction: "Stage all changes with `git add -A`, then commit with your message. Do not modify any files."

### 2. Add `gitDiffStat` and `gitDiff` helpers to `internal/orchestrator/implementation.go`

Small helpers alongside the existing `gitStatusDirty`:

- `gitDiffStat(ctx, dir) (string, error)` — runs `git diff --stat`
- `gitDiff(ctx, dir) (string, error)` — runs `git diff`

These provide the context the commit agent needs to write an informed message.

### 3. Modify `commitAndPushRepos` to spin up a commit session

Replace the direct `gitStageAndCommit` call with:

```go
if dirty, _ := gitStatusDirty(ctx, sess.WorktreePath); dirty {
    slog.Warn("agent left uncommitted changes in worktree; spinning up commit session",
        "repo", repo, "worktree", sess.WorktreePath, "session_id", sess.SessionID)
    if commitErr := s.commitViaAgent(ctx, sess.WorktreePath); commitErr != nil {
        slog.Error("agent commit session failed, falling back to generic message",
            "repo", repo, "error", commitErr)
        // Fallback to the old hardcoded message so we never leave uncommitted work
        if fallbackErr := gitStageAndCommit(ctx, sess.WorktreePath, "chore: commit residual changes before push"); fallbackErr != nil {
            slog.Error("failed to commit residual changes", "repo", repo, "error", fallbackErr)
        }
    }
}
```

### 4. Add `commitViaAgent` method to `ImplementationService`

```go
func (s *ImplementationService) commitViaAgent(ctx context.Context, worktreePath string) error {
    // 1. Gather diff context
    stat, err := gitDiffStat(ctx, worktreePath)
    if err != nil { return err }
    diff, err := gitDiff(ctx, worktreePath)
    if err != nil { return err }

    // 2. Build commit config from s.cfg
    commitCfg := adapter.CommitConfig{Strategy: "single", MessageFormat: "ai-generated"}
    if s.cfg != nil {
        commitCfg.MessageFormat = string(s.cfg.Commit.MessageFormat)
        commitCfg.MessageTemplate = s.cfg.Commit.MessageTemplate
    }

    // 3. Build prompt
    systemPrompt := buildCommitMessagePrompt(stat, diff, commitCfg)

    // 4. Start session (no push, no foreman, minimal timeout)
    opts := adapter.SessionOpts{
        Mode:          adapter.SessionModeAgent,
        WorktreePath:  worktreePath,
        SystemPrompt:  systemPrompt,
        UserPrompt:    "Commit the staged changes now.",
        CommitConfig:  commitCfg,
        AllowPush:     false,
    }
    sess, err := s.harness.StartSession(ctx, opts)
    if err != nil { return err }
    defer sess.Abort(ctx)

    // 5. Forward events (fire-and-forget for log visibility)
    go s.forwardEvents(ctx, sess.Events(), "", "")

    // 6. Wait for completion with a bounded timeout
    commitCtx, cancel := context.WithTimeout(ctx, 2*time.Minute)
    defer cancel()
    return sess.Wait(commitCtx)
}
```

Add a "Commit Message Format" subsection after the strategy description so the agent knows the format convention during implementation too. This is a secondary fix — the primary change is the safety-net commit session.

## Risks

- **Agent session overhead**: Spinning up a full agent session for a commit adds latency. Mitigated by the 2-minute timeout and fallback path.
- **Agent modifies files**: The prompt explicitly says "Do not modify any files" but agents can ignore instructions. Mitigated by this being a last-resort safety net — the implementation is already done and reviewed.
- **Harness unavailability**: If the harness can't start a session (binary not found, auth issue), the fallback ensures we still commit. The existing behavior is preserved as the fallback.
